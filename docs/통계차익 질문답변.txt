<질문>

지금 통계차익 기능을 만들고 있는데 데이터는 일봉은 3년 이상 데이터가 있고, 1분봉은 26년 1월부터만 있어 근데 통계차익 후보를 발굴할 때 일봉을 사용하면 종가가 튀는 종목들은 노이즈가 많잖아. 그래서 실제는 통계적으로 유의하지 않은데 포함될 수도 있는거고. 내가 통계차익 보는 기간은 짧으면 1주일에서 길면 반년 이상이기도 한데, 어떻게 해야할지 모르겠어. 전문가들은 
발굴할 때 어떤 데이터를 활용해? 일봉? 분봉? 하이브리드?

<답변>
핵심은 “후보 발굴”과 “실제 진입 신호”를 같은 데이터로 처리하지 않는 것입니다. 일봉만 쓰면 표본은 길지만 종가 노이즈가 있고, 분봉만 쓰면 미시구조 노이즈와 짧은 히스토리 문제가 생깁니다.
자료 기준으로 보면 전통적인 페어 트레이딩 연구는 일봉으로 후보를 만들었고, 최근/고빈도형 연구는 분봉을 쓰되 미시구조 노이즈를 별도 처리합니다. 따라서 “일봉 vs 분봉”이 아니라 역할 분리가 맞습니다.
결론부터 말하면 하이브리드가 맞습니다.
다만 역할을 나눠야 합니다.
후보 발굴은 일봉 중심 + 종가 노이즈 보정,
실제 진입·청산·체결 검증은 분봉 중심이 가장 현실적입니다.
________________________________________
1. 전문가들은 보통 어떻게 보나
공개 연구 기준으로 보면, 전통적인 페어 트레이딩은 일봉 데이터로 후보를 발굴한 사례가 많습니다. 대표적인 Gatev, Goetzmann, Rouwenhorst 연구도 1962~2002년 일봉 데이터로 가격 움직임이 가까운 종목쌍을 찾았습니다. 
반면 최근 고빈도형 통계차익 연구는 분봉·고빈도 데이터를 활용합니다. 예를 들어 분 단위 가격으로 S&P500 구성 종목의 페어 트레이딩을 분석한 연구도 있습니다. 
하지만 분봉을 많이 쓴다고 무조건 좋은 것은 아닙니다. 고빈도 가격에는 bid-ask bounce, 가격 단위, 거래 크기 차이 같은 market microstructure noise가 강하게 섞입니다. 특히 유동성이 낮은 종목일수록 noise-to-signal 비율이 커질 수 있다는 연구가 있습니다. 
그래서 실무적으로는 보통 이렇게 나눕니다.
목적	권장 데이터
후보군 1차 발굴	일봉, 30분봉, 60분봉
장기 관계 확인	일봉
최근 관계 유지 여부 확인	5분봉~30분봉
실제 진입·청산 시그널	1분봉, 호가, 체결
체결 가능성·슬리피지 검증	1분봉 + 호가/체결

2. 지금 상황에서는 “일봉만”도 “분봉만”도 위험함
현재 데이터 구조가 이렇습니다.
데이터	장점	문제
3년 이상 일봉	표본 기간이 길다. 반년 이상 관계 확인 가능	종가 튐, 단일 종가 왜곡, 장마감 동시호가 노이즈
2026년 1월 이후 1분봉	실제 매매 구간에 가깝다. 체결·괴리 확인 가능	기간이 짧다. 특정 장세만 반영될 수 있음
따라서 분봉만으로 후보를 발굴하면 과최적화 위험이 큽니다.
2026년 1월 이후의 시장 상태에서만 잘 맞는 페어가 뽑힐 수 있습니다.
반대로 일봉 종가만으로 후보를 뽑으면 가짜 페어가 섞일 수 있습니다.
말씀하신 것처럼 종가가 튀는 종목은 실제 평균회귀 관계가 없는데도 통계적으로 좋아 보일 수 있습니다.
________________________________________
3. 추천 구조
1단계: 일봉으로 넓게 발굴하되, 종가를 그대로 믿지 않기
일봉 후보 발굴 시 raw close만 쓰지 않는 게 좋습니다.
가능하면 아래 순서로 가격을 만드세요.
1.	일별 VWAP = 거래대금 / 거래량 
2.	종가 
3.	HLC3 = (고가 + 저가 + 종가) / 3 
4.	OHLC4 = (시가 + 고가 + 저가 + 종가) / 4 
가장 좋은 것은 일별 VWAP입니다.
종가 한 점보다 하루 전체 거래가 반영되기 때문에 종가 튐에 덜 민감합니다.
후보 발굴 때는 close 기준과 VWAP 기준을 둘 다 돌려서, 둘 다 통과하는 페어만 남기는 방식이 좋습니다.
________________________________________
2단계: 종가 튐 필터를 넣기
종가 노이즈가 큰 종목은 애초에 후보에서 제외하는 게 낫습니다.
예시 필터는 이런 식입니다.
abs(종가 / VWAP - 1) > 0.5% 또는 1.0% 인 날이 많은 종목 제외
또는 분봉이 있는 2026년 이후 구간에서는:
abs(종가 / 마지막 10분 VWAP - 1) > 기준값
이런 날이 자주 나오는 종목은 통계차익 후보로 부적합합니다.
특히 한국 주식에서는 장마감 동시호가, 저유동성 종목, 특정 수급성 종목에서 종가 왜곡이 꽤 생길 수 있습니다. 이건 확실한 일반론이고, 종목별 기준값은 백테스트로 잡아야 합니다.
________________________________________
3단계: 일봉 후보를 분봉으로 재검증
일봉에서 후보가 나왔다고 바로 쓰면 안 됩니다.
반드시 2026년 1월 이후 1분봉으로 아래를 확인해야 합니다.
검증 항목	의미
분봉 스프레드도 평균회귀하는가	일봉에서만 우연히 맞는지 확인
장중 괴리 발생 후 실제로 닫히는가	매매 가능성 확인
진입 신호가 장마감에만 생기는가	종가 노이즈 후보 제거
스프레드가 호가 스프레드보다 충분히 큰가	수수료·슬리피지 감당 가능 여부
괴리 발생 시 양쪽 종목 모두 체결 가능한가	실제 운용 가능성
즉, 일봉은 “후보 발견용”,
**분봉은 “실전 사용 가능성 검증용”**입니다.
________________________________________
4. 기간별로 데이터 주기를 다르게 써야 함
통계차익을 보는 기간이 1주일~반년 이상이라고 했으니, 하나의 주기로 통일하면 안 됩니다.
예상 보유 기간	후보 발굴 데이터	시그널 데이터
수분~당일	1분봉, 5분봉	1분봉, 호가
1일~1주	5분봉, 30분봉, 일봉	1분봉~5분봉
1주~1개월	일봉 + 30분봉	5분봉~30분봉
1개월~6개월	일봉 중심	일봉 + 장중 체결 검증
6개월 이상	일봉, 주봉	일봉
현재 목적이 “짧으면 1주일, 길면 반년 이상”이면, 후보 발굴은 일봉/30분봉 중심이 맞습니다.
1분봉은 후보 발굴보다 진입 타이밍, 리스크 확인, 체결 가능성 검증에 더 적합합니다.
________________________________________
5. 중요한 건 p-value보다 “관계의 안정성”
통계차익 후보를 많이 돌리면, 우연히 p-value가 좋아 보이는 조합이 반드시 나옵니다. 여러 전략·파라미터를 대량 테스트할 때 false discovery, data snooping 문제가 생긴다는 연구들이 있습니다. 
그래서 후보 발굴 기준은 단순히:
cointegration p-value < 0.05
이러면 부족합니다.
최소한 아래 조건을 같이 봐야 합니다.
1. 같은 업종/테마/지수/ETF 구성 관계가 있는가
2. 일봉 기준 cointegration 또는 spread 안정성이 있는가
3. 3년 전체가 아니라 최근 6개월~1년에서도 유지되는가
4. 2026년 이후 분봉에서도 스프레드가 무너지지 않았는가
5. half-life가 내 보유 기간과 맞는가
6. 거래대금, 스프레드, 체결 가능성이 충분한가
7. 특정 이벤트 한두 번 때문에 통계가 좋아진 것은 아닌가
동적 cointegration 연구에서도 평균회귀 속도와 half-life를 이용해 lookback window를 정하는 접근이 사용됩니다.

6. 내가 만들면 이렇게 설계할 것 같음
A. 후보 발굴용 가격 3개 생성
각 종목마다 아래 3개 시계열을 만듭니다.
P_close = 수정종가
P_vwap_daily = 거래대금 / 거래량
P_robust = close와 vwap의 괴리가 큰 날을 보정한 가격
P_robust 예시:
if abs(close / vwap - 1) > threshold:
    price = vwap
else:
    price = close
일봉에 거래대금/거래량이 있으면 이 방식이 꽤 실용적입니다.
________________________________________
B. 후보 발굴은 3개 기간으로 나눔
장기: 3년
중기: 1년
단기: 3~6개월
각 기간에서 모두 나쁘지 않은 페어만 남깁니다.
예를 들어:
3년 기준: 관계 존재
1년 기준: 관계 유지
3~6개월 기준: 최근에도 깨지지 않음
이렇게 해야 과거에는 좋았지만 최근에는 깨진 페어를 줄일 수 있습니다.
________________________________________
C. 분봉으로 최종 필터
일봉 후보 중에서 2026년 1월 이후 1분봉으로 다시 봅니다.
1. 1분봉 spread의 평균회귀성
2. 장중 z-score 발생 빈도
3. 진입 후 평균 회귀 시간
4. 손절 도달 빈도
5. 실제 bid/ask 기준 기대수익
여기서 중요한 것은 분봉 close가 아니라 bid/ask 또는 체결 가능 가격 기준입니다.
분봉 종가로 수익이 나도 실제 호가 스프레드 때문에 사라질 수 있습니다.
________________________________________
7. 실전 기준으로는 이런 구조가 좋음
후보 발굴:
- 일봉 3년
- 일봉 VWAP/종가 둘 다 사용
- 업종/테마/ETF 구성 관계 우선
- cointegration, correlation, spread 안정성 체크

후보 검증:
- 최근 6개월 일봉
- 2026년 이후 1분봉
- 종가 튐 필터
- half-life 확인

실제 매매:
- 1분봉 또는 5분봉 z-score
- 호가 기준 expected edge
- 체결 가능 수량 확인
- 장마감 근처 신호는 별도 취급

최종 결론
지금 데이터 구조에서는 하이브리드가 정답에 가깝습니다.
다만 비중은 이렇게 두는 게 좋습니다.
후보 발굴: 일봉 70%, 분봉 30%
실제 매매 신호: 분봉 70%, 일봉 30%
정확히 말하면:
일봉 = 구조적 관계를 찾는 데이터
분봉 = 그 관계가 실제 매매 가능한지 확인하는 데이터
종가가 튀는 문제는 일봉을 버릴 이유가 아니라, 일봉 가격을 robust하게 만드는 이유입니다.
가장 추천하는 방식은:
1. 일봉 VWAP 기반으로 1차 후보 발굴
2. 수정종가 기반 결과와 교차 확인
3. 종가/VWAP 괴리 큰 종목 제거
4. 최근 6개월 일봉으로 재검증
5. 2026년 이후 1분봉으로 실전 가능성 검증
6. 최종 진입은 1분봉/호가 기준
이렇게 가면 일봉의 장기 표본 장점과 분봉의 실전성 장점을 둘 다 살릴 수 있습니다.
