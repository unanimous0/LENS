# 종목차익 (Stock Arbitrage)

주식선물 베이시스를 활용한 개별종목 차익거래 화면.
실시간 데이터는 Rust 서비스에서 수신 (상세: [realtime-service.md](realtime-service.md)).

## 목적

주식 현물과 주식 선물의 괴리(베이시스)가 발생했을 때:
1. **매수차익 / 매도차익 포지션**을 잡기 위한 실시간 베이시스 모니터링
2. **ETF LP 관점에서**, ETF 매도 + 바스켓(PDF) 현물 매수 헤지 시 특정 종목의 선물이 현물보다 유리한 경우를 포착하여 기대수익을 계산

## 데이터 소스

### 마스터 데이터 (현물 ↔ 선물 매핑)

LS API t8401(목록) + t8402(상세)로 조회하여 `data/futures_master.json`에 저장.

- **250개 기초자산**, 근월/원월 각각 매핑
- 포함 데이터: 현물코드, 선물코드, 종목명, 만기일, 잔존일수, 거래승수
- **자동 갱신**: 서버 시작 시 근월물 만기일 체크 → 지났으면 LS API 재호출
- **내부망**: 외부망에서 만든 JSON 파일을 압축에 포함 (나중에 사내 서버 마스터 조회 가능 시 대체)
- API: `GET /api/arbitrage/master`, `POST /api/arbitrage/master/refresh`

### 실시간 데이터 (WebSocket)

| 메시지 | 용도 | 포함 데이터 |
|--------|------|------------|
| `stock_tick` | 현물 체결 | 가격, 체결량, 누적거래량 |
| `futures_tick` | 선물 체결 | 가격, 기초자산가, 시장베이시스(=선물-현물) |
| `etf_tick` | ETF 체결 | 가격, NAV, 괴리3종(현재가/bid/ask 기준) |

### 이론가 계산

```
이론선물가 = 현물가 × (1 + r × T/365) - 배당현가
이론베이시스 = 이론선물가 - 현물가
베이시스갭 = 시장베이시스 - 이론베이시스
```

- **금리(r)**: Rust 메모리에 저장, 프론트엔드 입력 → Backend API → Rust REST API
- **배당**: 복잡 — DART/거래소 API(공시) + 과거 배당 패턴(예상) 조합. 추후 별도 설계.
- **만기(T)**: 마스터 데이터의 만기일에서 자동 계산
- 내부망 사내 서버의 이론가(Index fl=12)는 금리/배당이 부정확하므로 직접 계산 필요

### 배당 데이터 (추후)

- DART / 한국거래소 API로 최근 배당 공시 확인
- 공시 미발표 시: 과거 분기/반기/연 배당 패턴으로 예상 배당 적용
- 시장이 배당을 미리 반영하므로, 공시 전이라도 예상치를 넣어야 이론가가 정확

### 보유 현황 (추후, 내부망 전용)

- KBM TCP 통신으로 조회 (`docs/internal_streaming_kbm.py` 참고)
- 031계정(019펀드), 052계정(018펀드) 잔고
- 최종 가능수량 = 담보가능수량 - 대여수량
- 보유펀드조회: 펀드세트 875 → 019875 + 018875의 장부/담보/대여 수량

### 스프레드

캘린더 스프레드 = **원월물 가격 - 근월물 가격** (현재 근월/원월만 사용)

---

## 틱 데이터 설계 원칙

- **틱에 포함**: 매 체결마다 바뀌는 값 (가격, 수량, 시장베이시스)
- **별도 조회**: 정적/반정적 데이터 (거래승수, 만기일, 배당, 종목명)
- **시장베이시스(basis)**: 순수 가격 차이 (선물-현물), bp 아님. bp 필요 시 프론트에서 변환
- **이론가/이론베이시스**: 금리+배당에 의존하므로, 해당 데이터가 Rust에 로드된 후 틱에 포함 예정
