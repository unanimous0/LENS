# 종목차익 (Stock Arbitrage)

주식선물 베이시스를 활용한 개별종목 차익거래 화면.
실시간 데이터는 Rust 서비스에서 수신 (상세: [realtime-service.md](realtime-service.md)).

## 목적

주식 현물과 주식 선물의 괴리(베이시스)가 발생했을 때:
1. **매수차익 / 매도차익 포지션**을 잡기 위한 실시간 베이시스 모니터링
2. **ETF LP 관점에서**, ETF 매도 + 바스켓(PDF) 현물 매수 헤지 시 특정 종목의 선물이 현물보다 유리한 경우를 포착하여 기대수익을 계산

---

## 기능 1: 실시간 베이시스 모니터링

주식선물 전체 종목(~2,300개)의 현물-선물 괴리를 실시간으로 표시하는 대시보드.

### 표시 데이터

| 항목 | 설명 | 소스 |
|------|------|------|
| 현물가 | 주식 현재가 | S3_/K3_ (실시간 체결) |
| 선물가 | 주식선물 현재가 | JC0 (실시간 체결) |
| 실제 베이시스 | 선물가 - 현물가 (또는 bp 환산) | 계산 |
| 이론 베이시스 | 금리 - 배당수익률 기반 이론가 괴리 | 입력/파일/DB |
| 괴리 (실제 - 이론) | 차익 기회 크기 | 계산 |
| 거래량 | 선물/현물 거래량 | 실시간 |
| 미결제약정 | 선물 미결제약정 | 실시간 |
| 배당수익률 | 종목별 예상 배당 | 입력/파일 |
| 잔존만기 | 선물 만기까지 남은 일수 | 계산 |

### 이론 베이시스 산출

```
이론선물가 = 현물가 × (1 + r × T/365) - 배당현가
이론베이시스 = 이론선물가 - 현물가
괴리 = 실제베이시스 - 이론베이시스
```

- `r`: 무위험이자율 (CD금리 등) — 수동 입력 또는 파일
- `T`: 잔존만기 (선물 만기일 - 오늘) — 자동 계산
- 배당: 종목별 예상 배당금/배당락일 — 엑셀 파일 또는 DB

### 입력 데이터 (정적)

| 데이터 | 형태 | 비고 |
|--------|------|------|
| 주식선물 종목 마스터 | LS API t8401 또는 엑셀 | 종목코드, 기초자산, 만기일, 거래승수 등 |
| 무위험이자율 | 수동 입력 | CD 91일물 등 |
| 배당 데이터 | 엑셀 파일 | 종목별 예상 배당금, 배당락일 |

### 실시간 데이터

Rust 서비스에서 수신하는 데이터 (TR 코드 상세: [ls-api.md](ls-api.md)):

| 데이터 | TR코드 | 비고 |
|--------|--------|------|
| 주식 체결 (KOSPI) | S3_ | 현물가 |
| 주식 체결 (KOSDAQ) | K3_ | 현물가 |
| 주식선물 체결 | JC0 | 선물가, 거래량, 미결제약정 |
| 주식선물 호가 | JH0 | 5호가 (선택) |

## 기능 2: ETF 차익 기회 계산

ETF LP가 헤지(ETF 매도 + PDF 현물 매수) 시, PDF 구성종목 중 선물이 현물보다 유리한 종목을 선물로 대체했을 때의 기대수익을 계산.

### 시나리오

1. ETF를 특정 호가에 매도
2. PDF(Portfolio Deposit File) 기준으로 바스켓 현물을 매수하여 헤지
3. PDF 구성종목 중 **베이시스가 낮은(선물이 싼) 종목**이 있다면:
   - 현물 대신 선물을 매수 → 현물-선물 가격 차이만큼 절약
4. 그 절약분 = ETF 호가를 더 공격적으로 제시할 수 있는 여유분

### 필요 데이터

| 데이터 | 형태 | 비고 |
|--------|------|------|
| ETF별 PDF | 엑셀 파일 (외부) / 거래소 데이터 (내부) | 구성종목, 비중, 수량 |
| 실시간 베이시스 | 기능 1에서 계산 | 종목별 현물-선물 괴리 |
| ETF 현재가/NAV | 실시간 (I5_) | ETF 괴리 확인 |

### 계산 로직 (개념)

```
PDF 구성종목 중 선물 상장 종목:
  각 종목별:
    절약금 = (현물가 - 선물가) × PDF 내 수량
    (베이시스 < 0 → 선물이 싸므로 절약 발생)

총 절약금 = sum(종목별 절약금)
ETF 1주당 기대수익 = 총 절약금 / ETF 설정단위
```

### 관점 선택 (미확정)

- **ETF 중심**: "이 ETF를 헤지할 때 선물 대체로 얼마나 절약?" → LP 업무 흐름에 자연스러움
- **개별종목 중심**: "이 종목 베이시스가 벌어졌는데, 어떤 ETF에서 활용?" → 기회 발견 관점

→ 추후 사용 패턴에 따라 결정

## 구현 단계

Rust 서비스 기반 인프라가 선행되어야 함 ([realtime-service.md](realtime-service.md) Phase 1).

### Phase A: 프론트엔드 종목차익 화면

Rust MockFeed 기반으로 UI 구현.

- [ ] 종목차익 페이지 생성 (탑 네비게이션에 추가)
- [ ] 실시간 베이시스 테이블 (종목코드, 현물가, 선물가, 베이시스, 이론베이시스, 괴리)
- [ ] 정렬/필터 (베이시스 크기순, 거래량 필터 등)
- [ ] 이론 베이시스 입력 UI (금리, 만기일)
- [ ] 배당 데이터 엑셀 업로드
- [ ] 주식선물 마스터 데이터 로드

### Phase B: 실제 시세 연동

Rust 서비스의 LS API / 사내 서버 연동 완료 후.

- [ ] Mock → 실시간 데이터 전환
- [ ] 2,300개 종목 대량 렌더링 최적화 (가상화 등)
- [ ] 베이시스 알림/하이라이트 (임계값 초과 시)

### Phase C: ETF 차익 기회 계산

- [ ] PDF 엑셀 파싱 (FastAPI)
- [ ] PDF 구성종목 ↔ 주식선물 매핑
- [ ] ETF별 차익 기회 계산 로직
- [ ] 프론트엔드 ETF 차익 화면
- [ ] ETF 호가별 기대수익 시뮬레이션
