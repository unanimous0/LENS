# 종목차익 (Stock Arbitrage)

주식선물 베이시스를 활용한 개별종목 차익거래 화면.
실시간 데이터는 Rust 서비스에서 수신 (상세: [realtime-service.md](realtime-service.md)).

## 목적

주식 현물과 주식 선물의 괴리(베이시스)가 발생했을 때:
1. **매수차익 / 매도차익 포지션**을 잡기 위한 실시간 베이시스 모니터링
2. **ETF LP 관점에서**, ETF 매도 + 바스켓(PDF) 현물 매수 헤지 시 특정 종목의 선물이 현물보다 유리한 경우를 포착하여 기대수익을 계산

## 데이터 소스

### 마스터 데이터 — 정적 매핑만 (가격 없음)

`data/futures_master.json` — LS API t8401(목록) + t8402(정적 정보)로 생성.

**포함**: 현물코드, 선물코드, 종목명, 만기일, 잔존일수, 거래승수, 시장구분(KOSPI/KOSDAQ), 스프레드코드
**미포함**: 가격, 거래량, 거래대금 (모든 가격은 실시간으로만)

- **갱신 시점**: 만기 지났을 때만 자동 갱신 (월 1회). 매일 갱신하지 않음.
- **승수 체크**: 매일 첫 로드 시 백그라운드로 t8402 승수 변경 확인 (분할/증자 대응)
- **내부망**: 외부망에서 만든 JSON 파일을 압축에 포함
- API: `GET /api/arbitrage/master` (즉시 응답), `POST /api/arbitrage/master/refresh` (강제 갱신)

### 가격 데이터 — 실시간 + 초기값

가격은 마스터에 저장하지 않음. 두 가지 경로로 제공:

**1. 실시간 (WebSocket 체결)**
| 메시지 | 소스 TR | 포함 데이터 |
|--------|---------|------------|
| `stock_tick` | S3_(코스피) / K3_(코스닥) | 가격, 체결량, 거래대금 |
| `futures_tick` | JC0 | 가격, 기초자산가, 시장베이시스 |
| `etf_tick` | S3_/K3_ (ETF) | 가격, NAV, 괴리 |

**2. 초기값 (REST, 서비스 시작 시)**
| TR | 대상 | 제공 데이터 | 비고 |
|----|------|------------|------|
| t8402 | 선물 250 + 스프레드 250 | 마지막 체결가, 거래량, 기초자산가 | `is_initial: true` |
| t1102 | 현물 250 | 마지막 체결가, 거래대금 | `is_initial: true` |

- t8402와 t1102는 **병렬 태스크**로 동시 실행 (TPS 풀이 별도)
- 실시간 체결이 먼저 오면 초기값은 무시 (`is_initial`이고 이미 값 있으면 skip)
- 실시간 체결이 없는 종목만 초기값(마지막 체결가)으로 커버
- 전체 완료까지 ~60-90초이지만, 활발한 종목은 1-2초 내 실시간으로 채워짐

### 스프레드 — 실시간 D코드 종목

캘린더 스프레드 = 근월-차월 스프레드 상품 (D코드). 계산값이 아닌 **실제 체결가**.

- 마스터에서 근월-차월 스프레드 코드 자동 매칭 (t8401 D코드 + front/back month 패턴)
- JC0로 실시간 구독 (고정 그룹, 월물 전환 시 변경 안 됨)
- 월물 전환해도 스프레드는 동일 (05-06 스프레드는 항상 05-06)

### 이론가 계산

```
이론선물가 = 현물가 × (1 + r × T/365) - 배당금
이론베이시스 = 이론선물가 - 현물가
베이시스갭 = 시장베이시스 - 이론베이시스
```

- **갭 계산 토글 (이론가 / 이론0원)**: 상단 토글 (`arbitrage.basisMode`, localStorage 유지).
  - **이론가** (기본): `갭 = 시장B − 이론B`. 시장이 이론으로 수렴한다 가정 → 단기 차익(엔트리/언와인드) 시점에 수익 추정.
  - **이론0원**: `갭 = 시장B − 0`. 만기까지 holding 가정 — 만기엔 이론B도 0으로 수렴하므로 시장베이시스 자체가 만기 수익. 이 모드에선 표시상 이론B 컬럼도 `0`으로 표시(이론가 컬럼은 실제 계산값 유지). 갭bp 분모는 두 모드 모두 현물가.
- **금리(r)**: 프론트엔드 상단 입력칸 (`stock-arbitrage.tsx`), localStorage에 저장 (`arbitrage.rate`), 디폴트 2.8%. commit-on-blur/Enter 패턴 — 타이핑 중엔 재계산 안 되고, 포커스 이동/엔터 시 한 번만 전체 행 재계산. 입력 취소 시 이전 값 복구. 실시간 체결과 독립이라 Rust 왕복 불필요.
- **배당**: 현재 `0`으로 고정. 배당 화면(추후)에서 배당금 + 배당기준일(또는 배당락일)을 조회해서 해당 월물에만 적용 예정. 공시 전 예상치는 과거 패턴으로 추정.
- **만기(T)**: `master.items[].front.days_left` (= LS API `jandatecnt`). 잔존일 = 만기일 − 오늘 (만기 당일 0). 선택 월물에 따라 front/back 전환.
- 내부망 사내 서버의 이론가(Index fl=12)는 금리/배당이 부정확하므로 직접 계산 필요

### 배당 데이터 (추후)

- DART / 한국거래소 API로 최근 배당 공시 확인
- 공시 미발표 시: 과거 분기/반기/연 배당 패턴으로 예상 배당 적용
- 시장이 배당을 미리 반영하므로, 공시 전이라도 예상치를 넣어야 이론가가 정확

### 보유 현황 (추후, 내부망 전용)

- KBM TCP 통신으로 조회 (`docs/internal_streaming_kbm.py` 참고)
- 031계정(019펀드), 052계정(018펀드) 잔고
- 최종 가능수량 = 담보가능수량 - 대여수량
- 보유펀드조회: 펀드세트 875 → 019875 + 018875의 장부/담보/대여 수량

### 스프레드

캘린더 스프레드 = **원월물 가격 - 근월물 가격** (현재 근월/원월만 사용)

## 데이터 설계 원칙

- **가격은 실시간만**: 마스터에 가격을 저장하지 않음. 모든 가격은 실시간 체결 또는 초기값(t8402/t1102)에서.
- **체결 없으면 0**: 오늘 체결 내역이 없는 종목은 공란. stale 값을 현재가처럼 보여주지 않음.
- **초기값은 보조**: `is_initial: true` 플래그로 실시간 체결이 이미 있으면 무시. 체결이 아직 없는 종목에만 적용.
- **정적 정보는 마스터**: 코드 매핑, 종목명, 만기일, 승수만. 갱신 주기 = 만기 시 (승수는 매일 백그라운드 체크).
- **시장베이시스(basis)**: 순수 가격 차이 (선물-현물), bp 아님. bp 필요 시 프론트에서 변환.
- **근월/원월 동일 로직**: 코드만 다르고 데이터 경로 완전 동일. 차이는 시장 유동성뿐.
