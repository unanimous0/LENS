# 구현된 기능 상세

## 대차 페이지 (`/lending`)

서브탭 구조로 6개 기능을 관리:
- **관리**: 차입 (구현 완료) · 상환 (미구현) · 대여 (구현 완료) · 대여상환 (미구현)
- **확인**: 대여가능확인 · 상환가능확인 (구현 완료)

---

## 차입 (서브탭)

대차내역.xls(예탁원 e-safe) 업로드 → 내부 탭 2개: **비용 분석** / **Rollover 3 확인**

### 입력 파일

| 파일 | 출처 | 주요 컬럼 |
|------|------|-----------|
| 대차내역.xls | 한국예탁결제원 e-safe | D:단축코드, E:종목명, F:수수료율, G:대차수량, H:대차가액, J:체결일, K:체결번호, M:상환만기일, O:대여자계좌, P:대여자명, AG:Rollover횟수 |

### 비용 분석

세 가지 관점으로 차입 비용 집계:

| 뷰 | 그룹 기준 | 표시 항목 |
|---|---|---|
| 차입처별 | 대여자계좌+대여자명 | 가중평균 수수료, 총 대차가액, 일일 비용 |
| 고비용 종목 | 수수료율 > 0.05% 종목 | 가중평균 수수료, 총 대차가액, 일일 비용 |
| 종목별 | 단축코드+종목명 | 가중평균 수수료, 총 대차가액, 일일 비용 |

- 가중평균 수수료 = `sum(수수료율 × 대차가액) / sum(대차가액)`
- 일일 비용 = `총 대차가액 × 가중평균수수료율 / 100 / 365`
- 모든 테이블 컬럼 정렬 지원 (클릭 토글)
- 테이블 max-height 10행, 초과 시 내부 스크롤
- 행 hover 시 상세 내역 tooltip (fixed position, viewport 경계 감지로 위/아래 방향 자동 전환)

### Rollover 상환 관리

AG열(Rollover 횟수) ≥ 3인 건은 M열(상환만기일)까지 필수 상환.

- 월별 드롭다운 필터 (기본: 오늘 기준 월) + 전체 보기
- 표시 정보: 종목코드, 종목명, 상환만기일(D-day), 대여자, 체결일, 체결번호, 대차수량, 대차가액, 수수료율
- 요약 카드: 선택 월 상환 대상/대차가액 + 전체 상환 대상/대차가액
- 엑셀 저장: 종목코드, 종목명, 대차수량, 대여자코드, 대여자명, 체결일, 체결번호, 상환만기일 (상환만기일 오름차순)

## 대여 (서브탭)

대여내역.xls(예탁원 e-safe) 업로드 → **수익 분석** (Rollover 없음)

차입과 동일한 e-safe 파일 포맷. `BorrowingPage` 컴포넌트를 `mode="lending"`으로 재사용.

### 입력 파일

| 파일 | 출처 | 비고 |
|------|------|------|
| 대여내역.xls | 한국예탁결제원 e-safe | 대차내역과 동일 34컬럼. 차입자계좌/차입자명을 상대방으로 사용 |

### 차입과의 차이점

| 항목 | 차입 | 대여 |
|------|------|------|
| 상대방 컬럼 | 대여자계좌/대여자명 | 차입자계좌/차입자명 |
| 관점 | 비용 (우리가 지불) | 수익 (우리가 수취) |
| 고비용/고수익 기준 | 수수료율 > 0.05% | 수수료율 ≥ 5% |
| Rollover 관리 | 있음 | 없음 |
| 라벨 | 차입처별 비용, 1D 비용 | 차입처별 수익, 1D 수익 |

### 수익 분석

세 가지 관점으로 대여 수익 집계:

| 뷰 | 내용 |
|---|---|
| 차입처별 수익 | 차입자(상대방 증권사)별 가중평균 수수료, 총 대차가액, 1D 수익 |
| 5% 이상 종목 | 수수료율 ≥ 5% 종목만 필터 |
| 종목별 수익 | 전 종목 수익 집계 |

## 대여가능확인 (서브탭)

개별 파일 5개 업로드 또는 폴더 경로 지정 → 자동 필터링 → 종목별 대여 가능 수량 + 펀드별 상세 내역 + 엑셀 저장

### 입력 파일 (개별 파일 또는 폴더 경로)

| 파일 | 폴더 패턴 | 내용 |
|------|-----------|------|
| 대여문의종목 | `대여문의종목*` | 종목코드, 종목명, 최대수량, 요율 |
| 5264 (원장RAW) | `5264*` | 펀드코드, 펀드명, 계정코드, 종목번호, 종목명, 잔고, 담보, 담보가능수량 등 (A~N열, 14개 컬럼) |
| 대여불가펀드 | `대여불가펀드*` | 제외할 펀드코드 목록 (파일 + 수동 입력 합산) |
| MM펀드 | `MM펀드*` | 제외할 MM펀드 코드 목록 |
| 상환예정내역 | `상환예정*` | D열: ISIN코드, J열: 대차수량 |

폴더 경로 모드: `file_resolver`가 폴더 내에서 패턴 매칭으로 파일 자동 탐색 (최근 수정일 우선, NFC/NFD 정규화 지원). 대여불가펀드는 파일 기반 + 프론트 수동 입력(기본값 7개: 187, 421, 422, 424, 446, 476, 836)을 합산.

### 처리 로직

1. 문의종목 기준으로 원장RAW 매칭
2. MM펀드(펀드코드 일치) 제외
3. 대여불가펀드(파일 + 수동 입력, 뒤3자리 또는 6자리 매칭) 제외
4. 상환예정 수량 차감 (계정코드 052 우선 → 담보가능수량 큰 순서대로 가능한 만큼 차감, 부족하면 다음 펀드로)
5. 합산 수량 0인 펀드 제외
6. 결과: 종목별 담보가능수량 / 상환예정 / 담보가능-상환 / 담보 / 대여가능수량 / 대여 + 펀드별 상세
7. 담보해제 필요 표시: 담보가능-상환 < 문의수량 && 담보 > 0인 종목에 오렌지 배지

### 종목코드 정규화 (대차 페이지 전반)

`backend/services/stock_code.py::normalize_stock_code`로 통일된 정규화. 8곳 적용 (lending_parser × 3, repayment_parser × 3, repayment_calculator 필터 × 2, borrowing_calculator × 1).

| 입력 | → 정규화 결과 | 비고 |
|---|---|---|
| `005930` | `005930` | 표준 6자리 |
| `A005930` | `005930` | 'A' 접두 (5264 / 사용자 입력) |
| `KR7005930003` | `005930` | ISIN |
| `5930` | `005930` | zfill 보정 |
| `'A005440      '` | `005440` | trailing whitespace strip |
| `0000J0` | `0000J0` | ETF 영문 포함 6자리 유지 |
| `KAM566000` | `KAM566000` | 9자리 선물코드 보호 (A trim 조건 6 < len ≤ 8) |

**원본 버그 (2026-05 발견)**: `parse_inquiry`가 `zfill(6)`만 했음 → `A005930`은 7자리라 zfill 무동작 → 원장의 `005930`과 매칭 실패. 마찬가지로 5264 Sheet1처럼 종목번호 셀에 trailing whitespace가 들어오면 이전 코드 `replace(r"^A","").zfill(6)`이 공백 안 떼서 매칭 실패 → 가능수량 0으로 표시되는 현상. 정규화 함수의 `strip().upper()`로 자동 해결.

사용자가 6자리/A접두/ISIN 어느 형식으로든 입력 가능. 매칭 양쪽이 같은 함수를 거치므로 누락 없음.

## 상환가능확인 (서브탭)

개별 파일 5개 업로드 또는 폴더 경로 지정 → 필터링 → 투 포인터 매칭 → 상환 계획 산출 + 엑셀 저장

### 입력 파일 (개별 파일 또는 폴더 경로)

| 파일 | 폴더 패턴 | 내용 |
|------|-----------|------|
| 5264 (오피스) | `5264*` | 펀드별 종목의 담보가능수량, 담보잡힌수량 |
| 대차내역 (예탁원) | `대차내역*` | 체결된 대차 건별 상세 (대여자, 수수료율, 대차수량 등) |
| 상환예정내역 | `상환예정*` | D열: ISIN코드, J열: 대차수량 |
| MM펀드 | `MM펀드*` | 제외할 MM펀드 코드 (대여가능확인과 공유) |
| 대여불가펀드 | `대여불가펀드*` | 제외할 펀드코드 (대여가능확인과 공유) |

폴더 경로 모드: 대여가능확인과 동일 (file_resolver, NFC/NFD 정규화).

### 처리 로직

1. MM펀드(펀드코드 일치) 제외
2. 대여불가펀드(suffix 3자리/6자리 매칭) 제외
3. 수동 필터 적용 (5264 제외조건 + 대차내역 제외조건)
4. 상환예정 수량 차감 (계정코드 052 우선 → 담보가능수량 큰 순서대로)
5. 상환 매칭: 종목별로 오피스(052 우선 → 담보가능수량 작은 순)과 예탁원(수수료율 높은 순 → 수량 작은 순) 투 포인터 매칭
6. `min(오피스 수량, 예탁원 수량)` 만큼 상환, 소진 시 다음 인덱스로 이동
7. 상환금액 = 기준가액 × 상환수량

### 필터링 (계산 전 적용)

- 5264: 펀드코드 제외(태그, 기본값 7개), 종목코드(태그), 종목명(태그)
- 대차내역: 운용사/증권사 제외(체크박스), 대여자(드롭다운+직접입력), 수수료율 이하 제외(기본 0.05), 체결일 제외(태그, 기본 오늘), 종목코드(태그), 종목명(태그)

### 결과

- 메인 테이블: 담보가능수량 / 상환예정 / 담보가능-상환 / 담보 / 합산 / 상환수량 / 상환금액 / 체결건수 / 최고수수료율
- 아코디언: 펀드별 담보가능수량 / 상환예정 / 담보가능-상환 / 담보 + 매칭 건별 상세
- 전량 차감 펀드: 상환예정으로 전량 소진된 펀드도 아코디언에 표시 (펀드명 포함)
- 담보해제 필요 표시: 상환수량 ≥ 담보가능-상환 && 담보 > 0인 종목에 오렌지 배지
- 엑셀 저장 (상환 내역, 종목별 합산, 상환 후 오피스/예탁원, 내부차입 추정)

## 배당 (`/dividends`)

배당 데이터 조회 + 다가오는 배당 시각화 + LENS-side 추정. Finance_Data DB가 SSOT, LENS는 read-only 소비자 + 추정 레이어.

### 메인 테이블 — 종목당 1행 (이벤트 뷰 → 종목별 뷰)

| 컬럼 | 설명 |
|------|------|
| No / 종목 / 정관 | 종목 메타 |
| **다가오는 배당** [배당락일·배당금·상태·구분·정정·원문] | 미래 (확정 + LENS 추정) |
| **최근 배당** [배당락일·배당금·상태·구분·정정·원문] | 과거 가장 최근 1건 |
| 수익률 (YYYY 예상) | 올해 calendar year `yield_pct` 합산 |

- **모든 컬럼 정렬 가능**, 빈 값 항상 맨 아래
- **가상화** (@tanstack/react-virtual) — 1,500+종목 가뿐
- 그룹 사이 vertical border로 다가오는/최근 시각 분리
- 다가오는 배당락일·배당금: 초록 텍스트 (text-up)
- 상태 배지: 확정(초록) / 예상(주황 + estimation_basis 호버 툴팁)
- 정관 호버 시 그리드 툴팁 (A=변경기입, B=미변경기입 + 추정 신뢰도)
- sticky 헤더 갭 fix (`borderCollapse: collapse` + bg-bg-primary 명시)

### 우측 디테일 패널 (종목 클릭 시)

- 종목 헤더 + BarChart 추이 (Inter 폰트로 막대 위 배당금 라벨)
- 카드 목록 — 그 종목 전체 이력
  - 좌측 세로줄: 확정=초록, 예상=주황 (3px)
  - 배당락일·D±N·구분·출처·상태 배지 + 풀 디테일
  - 정정공시 이력 grid (버전·배당금·배당락일·기준일·공시일·원문)
- 종목 재클릭 또는 × 버튼으로 닫힘

### 차트 (좌·우 별개 패널로 분리)

- **다가오는 배당 (90일 내)** Scatter — y=배당수익률(%), x=D-day, 모든 점 균일 크기, 모두 초록
  - hover 툴팁: 종목명/코드 + grid (배당락일/D-day/배당금/수익률/구분/상태)
  - 미선택 시 자동 폴백 (정렬된 첫 행)
- **배당 추이** BarChart — 선택 종목 최근 10건 (확정=초록, 예상=주황)

### 데이터 소스 우선순위 (`backend/routers/dividends.py`)
1. `data/dividends.json` (Finance_Data export — 운영 시 우선)
2. `data/dividends_mock.json` (개발용 mock)

**캐싱**: 날짜 + mtime 기반 자동 reload. ETL 파일 갱신 시 즉시 반영.

**종목명 fallback**: API의 `name` (Finance_Data DART corp_name) → 마스터 → 코드 순.

### LENS-side 추정 레이어 (`backend/services/dividend_estimator.py`)

DB는 사실(DART 확정)만, 추정은 매 요청 동적 계산.

- 각 (code, period)별 가장 최근 확정 배당 → 1년 / 2년 후 동일 날짜 후보
- 금액: 작년 동일 period 그대로 (보수적)
- horizon: today + 12개월
- 마지막 배당 > 730일 전이면 skip (회사가 배당 중단했을 가능성)
- 결과: `confirmed=False`, `source='ESTIMATE'`, 합성 ID `est-{code}-{period}-{year}`
- 프론트가 노란빛 + "예상" 배지로 자동 구분

### 종목차익 페이지 통합

행별로 `(today ≤ ex_date ≤ 만기일)` 매칭 배당 합산 → 이론가 식의 `D` 자리에 자동 반영. 추정 배당도 포함됨.

## 데이터 어댑터 (Rust realtime 서비스)

실시간 데이터는 Python backend가 아닌 **Rust realtime 서비스 (port 8200)** 가 담당. 2026-04 backend에서 분리됨 (성능/안정성).

`feed/` 디렉토리의 어댑터:
- **`ls_api`**: LS증권 OpenAPI WebSocket (S3_/K3_/JC0/JH0/H1_/HA_/I5_/VI_) + REST (t1102/t8402/t1405/t1404). 외부망 가정.
- **`internal`**: 사내 거래소 수신 서버 (내부망 가정)
- **`mock`**: 가짜 시세 (개발용, 시장 시간 무관)

프론트엔드 NetworkToggle → `POST /mode/{mode}` (Rust) → 어댑터 교체 + cache clear + 신규 spawn. 프론트는 단일 `/ws/market` WebSocket 유지.

## 공통: 엑셀 읽기 (excel_reader.py)

모든 엑셀 파싱에서 공통 모듈 사용:
1. openpyxl (.xlsx) 시도
2. 실패 시 xlrd (.xls) 시도
3. 둘 다 실패 시 xlwings fallback (Windows + Excel 필요, 보안 프로그램 우회)

내부망 보안 프로그램이 zip 해제를 차단하면 openpyxl이 실패 → xlwings로 자동 전환.
