# 구현된 기능 상세

## 대차 페이지 (`/lending`)

서브탭 구조로 6개 기능을 관리:
- **관리**: 차입 (구현 완료) · 상환 · 대여 · 대여상환 (미구현)
- **확인**: 대여가능확인 · 상환가능확인 (구현 완료)

---

## 차입 (서브탭)

대차내역.xls(예탁원 e-safe) 업로드 → 내부 탭 2개: **비용 분석** / **Rollover 3 확인**

### 입력 파일

| 파일 | 출처 | 주요 컬럼 |
|------|------|-----------|
| 대차내역.xls | 한국예탁결제원 e-safe | D:단축코드, E:종목명, F:수수료율, G:대차수량, H:대차가액, J:체결일, K:체결번호, M:상환만기일, O:대여자계좌, P:대여자명, AG:Rollover횟수 |

### 비용 분석

세 가지 관점으로 차입 비용 집계:

| 뷰 | 그룹 기준 | 표시 항목 |
|---|---|---|
| 차입처별 | 대여자계좌+대여자명 | 가중평균 수수료, 총 대차가액, 일일 비용 |
| 고비용 종목 | 수수료율 > 0.05% 종목 | 가중평균 수수료, 총 대차가액, 일일 비용 |
| 종목별 | 단축코드+종목명 | 가중평균 수수료, 총 대차가액, 일일 비용 |

- 가중평균 수수료 = `sum(수수료율 × 대차가액) / sum(대차가액)`
- 일일 비용 = `총 대차가액 × 가중평균수수료율 / 100 / 365`
- 모든 테이블 컬럼 정렬 지원 (클릭 토글)
- 테이블 max-height 10행, 초과 시 내부 스크롤
- 행 hover 시 상세 내역 tooltip (fixed position, viewport 경계 감지로 위/아래 방향 자동 전환)

### Rollover 상환 관리

AG열(Rollover 횟수) ≥ 3인 건은 M열(상환만기일)까지 필수 상환.

- 월별 드롭다운 필터 (기본: 오늘 기준 월) + 전체 보기
- 표시 정보: 종목코드, 종목명, 상환만기일(D-day), 대여자, 체결일, 체결번호, 대차수량, 대차가액, 수수료율
- 요약 카드: 선택 월 상환 대상/대차가액 + 전체 상환 대상/대차가액
- 엑셀 저장: 종목코드, 종목명, 대차수량, 대여자코드, 대여자명, 체결일, 체결번호, 상환만기일 (상환만기일 오름차순)

## 대여가능확인 (서브탭)

엑셀 파일 업로드 → 자동 필터링 → 종목별 대여 가능 수량 + 펀드별 상세 내역 + 엑셀 저장

### 필수 시트 (시트 이름 정확히 일치해야 함)

| 시트명 | 내용 |
|--------|------|
| 문의종목 | 종목코드, 종목명, 최대수량, 요율 |
| 원장RAW | 펀드코드, 펀드명, 계정코드, 종목번호, 종목명, 잔고, 담보, 담보가능수량 등 (A~N열, 14개 컬럼) |
| MM펀드 | 제외할 MM펀드 코드 목록 |
| 상환예정내역 | D열: ISIN코드, J열: 대차수량 |

### 대여불가펀드

엑셀 시트 불필요. 프론트엔드 필터 패널에서 펀드코드 3자리(뒤3자리 매칭) 또는 6자리(정확 일치)를 태그로 관리 (기본값 7개: 187, 421, 422, 424, 446, 476, 836)

### 처리 로직

1. 문의종목 기준으로 원장RAW 매칭
2. MM펀드(펀드코드 일치) 제외
3. 대여불가펀드(프론트에서 입력한 뒤3자리 기준) 제외
4. 상환예정 수량 차감 (계정코드 052 우선 → 담보가능수량 큰 순서대로 가능한 만큼 차감, 부족하면 다음 펀드로)
5. 합산 수량 0인 펀드 제외
6. 결과: 종목별 담보가능수량 / 상환예정 / 담보가능-상환 / 담보 / 합산 / 대여 + 펀드별 상세

### 종목코드 정규화

- 문의종목: `011790` (6자리 기준)
- 원장RAW: `A011790` → A 제거
- 상환예정: `KR7011790004` → [3:9] 추출

## 상환가능확인 (서브탭)

오피스(5264) + 예탁원(대차내역) 2개 파일 업로드 → 투 포인터 매칭 → 상환 계획 산출 + 엑셀 저장

### 입력 파일

| 파일 | 출처 | 내용 |
|------|------|------|
| 5264.xlsx | 사내 오피스 | 펀드별 종목의 담보가능수량 |
| 대차내역.xls | 한국예탁결제원 e-safe | 체결된 대차 건별 상세 (대여자, 수수료율, 대차수량 등) |

### 매칭 로직

1. 종목별로 오피스 담보가능수량(작은 순)과 예탁원 대차수량(수수료율 높은 순) 투 포인터 매칭
2. `min(오피스 수량, 예탁원 수량)` 만큼 상환, 소진 시 다음 인덱스로 이동
3. 상환금액 = 기준가액 × 상환수량

### 필터링 (계산 전 적용)

- 5264: 펀드코드 제외(태그), 종목코드(태그), 종목명(태그)
- 대차내역: 운용사/증권사 제외(체크박스), 대여자(드롭다운+직접입력), 수수료율 N% 이하 제외, 체결일 제외(태그, 기본 오늘), 종목코드(태그), 종목명(태그)
- 모든 필터는 정확히 일치 기준 (운용사/증권사 체크박스만 부분 일치: "운용"/"증권" 포함)

### 결과

- 상환 매칭 내역 (아코디언: 종목별 합산 → 펀드+체결건 상세)
- 미상환 내역 (예탁원에서 상환되지 못한 건)
- 내부차입 추정 (오피스에 있지만 예탁원에 없는 종목)
- 엑셀 저장 (상환 내역, 종목별 합산, 상환 후 오피스/예탁원, 내부차입 추정)

## 실시간 시세 (`/market`)

WebSocket 기반 실시간 ETF/선물 데이터 스트리밍.
- ETF 괴리 히트맵 (bp 기준 색상)
- ETF 실시간 현황 테이블 (현재가, NAV, 괴리, 거래량)
- 선물 베이시스 테이블 (선물가, 현물가, 베이시스)
- 네트워크 모드 전환: 내부망/외부망/Mock (런타임 전환, 재시작 불필요)

---

## 데이터 어댑터 패턴

`MarketDataAdapter` ABC로 데이터 소스 추상화:
- `MockAdapter`: 개발용 랜덤 시세 (8 ETF + 3 선물, 1초 간격)
- `InternalAdapter`: 내부망 (미구현)
- `ExternalAdapter`: 외부망 (미구현)

프론트엔드 토글 → `POST /api/network/mode/{mode}` → 어댑터 교체 → 스트리밍 재시작

---

## 공통: 엑셀 읽기 (excel_reader.py)

모든 엑셀 파싱에서 공통 모듈 사용:
1. openpyxl (.xlsx) 시도
2. 실패 시 xlrd (.xls) 시도
3. 둘 다 실패 시 xlwings fallback (Windows + Excel 필요, 보안 프로그램 우회)

내부망 보안 프로그램이 zip 해제를 차단하면 openpyxl이 실패 → xlwings로 자동 전환.

---

## 공통: 엑셀 저장 서식 (excel.ts)

모든 엑셀 저장 시 `formatSheet()` 적용:
- 정수: `#,##0` (천단위 쉼표)
- 소수점: `#,##0.00` (둘째자리)
- 컬럼 너비: 내용 기준 자동 맞춤 (한글 2칸, 영문/숫자 1칸)
